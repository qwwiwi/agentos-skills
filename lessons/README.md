# Уроки AgentOS

Боевые уроки для интенсива AgentOS. Каждый урок – пошаговое прохождение реальной задачи, от пустой консоли до работающей системы. После урока у тебя в руках не теория, а живой результат.

## Структура

Каждый урок лежит в своей папке и содержит:

- `README.md` – обзор урока, цель, outcomes, prerequisites, оценка времени
- `01-...md`, `02-...md`, ... – пронумерованные шаги, читаешь по порядку
- Последний шаг обычно `troubleshooting.md` – типичные ошибки и решения

Шаги атомарны: можешь прерваться на любом, вернуться позже, продолжить без потери контекста.

## Доступные уроки

### [lesson-3-agents-with-gbrain](./lesson-3-agents-with-gbrain/)

Поднимаем три AI-агента (coder, marketer, sales) и подключаем их к общему gbrain (Second Brain). Через 90 минут у тебя три рабочих Claude Code агента, которые шлют друг другу задачи и видят общую базу знаний.

**Стек:** Claude Code, public-architecture-claude-code, public-gbrain-agentos, MCP, Postgres+pgvector.

**Outcome:** три агента ↔ shared memory ↔ swarm.notify между ними.

## Зачем эти уроки

Один агент в Claude Code – удобно. Несколько агентов с общей памятью – принципиально другая мощь: один пишет код, второй думает про маркетинг, третий ведёт продажи, и они видят работу друг друга.

Документация по каждому компоненту есть, но собрать всё вместе – нетривиально. Эти уроки закрывают «склейку»: что с чем подключить, какие порядки шагов, где типично ломается.

## Как читать

1. Открой `README.md` урока – пойми, нужен ли он тебе сейчас
2. Проверь prerequisites – не начинай если чего-то нет
3. Иди по шагам по порядку, не пропускай smoke-тесты
4. Если застрял – загляни в `09-troubleshooting.md` (или аналог)
5. После урока – глянь раздел «что дальше» в README, там идеи для расширения

## Обратная связь

Если урок не сработал на твоей системе – открой issue в репо `agentos-skills` с конкретным шагом и логом ошибки. Уроки обновляются по реальным сбоям учеников.
