# Частые вопросы

Подборка вопросов, которые приходят чаще всего на старте. Если ответа на свой не нашёл – открой issue в репозитории.

## 1. Сколько стоит запустить 3 агентов с gbrain?

Базовая ежемесячная стоимость стека:

– Anthropic Max: $200/мес – одна подписка покрывает все три агента (один Claude Code account)
– VPS под gbrain (Hetzner CPX42 или аналог): ~12€/мес ≈ $13
– Telegram, GitHub, Let's Encrypt: бесплатно
– Опционально – домен: ~$10/год (≈ $1/мес)

**Итого: ~$215/мес** при условии что Max подписка одна на всех. Если хочешь экономить – см. следующий вопрос про Pro.

## 2. Можно ли использовать Anthropic Pro вместо Max?

Да, технически работает. Подвохи:

– Pro ($20/мес) даёт меньший месячный лимит запросов в Claude Code (limit перезагружается каждые 5 часов)
– При активной работе трёх агентов лимит выгорает за несколько часов – дальше упираешься в ожидание или переходишь на API per-token
– API ключ доступен отдельно – можно положить как fallback, но это уже per-request биллинг ($3–15/мес на агента в зависимости от нагрузки)

Если ты только начинаешь и хочешь попробовать – Pro + API key на запас приемлемо. Когда выйдешь на постоянный workflow – Max окупится.

## 3. Зачем нужен gbrain если каждый агент уже умеет помнить?

Каждый агент в Claude Code имеет локальную память: `MEMORY.md`, `recent.md`, hooks. Этого достаточно для одиночного агента. Но как только агентов становится двое и больше – появляются проблемы:

– **Shared knowledge**: маркетер написал пост, кодер должен знать о нём (контекст для лендинга), sales должен иметь черновик в Telegram. Без gbrain ты вручную копируешь между workspace'ами
– **Swarm coordination**: координатор делегирует задачу кодеру через `swarm.notify` – gbrain ведёт outbox с retry, гарантирует доставку
– **Семантический поиск по истории**: «что мы решали про pricing три месяца назад?» – локальная память не индексирует через embeddings, gbrain даёт pgvector recall

Если планируешь работать с одним агентом – gbrain не обязателен. Двое и больше – нужен.

## 4. Можно ли запустить агентов без Telegram?

Да. `dashi-plugin-claude-code` (Telegram bridge) – опциональный компонент. Можно использовать агентов:

– Прямо в терминале (`claude` запускается, ты пишешь в TUI)
– Через VS Code расширение Claude Code
– Через web интерфейс Claude.com (с подключённым workspace)

Telegram удобен потому что:

– Работает с телефона
– Каждому агенту свой bot – чёткая изоляция каналов
– История чата = бэкап общения с агентом

Но если телефон тебе не нужен – пропускай эту часть стека.

## 5. Что если у меня нет своего VPS?

Варианты:

– **gbrain локально через Docker**: works, репозиторий `public-gbrain-agentos` поддерживает docker-compose. Но Telegram webhook требует публичный HTTPS endpoint, для локального запуска понадобится туннель (ngrok / Cloudflare Tunnel / Tailscale Funnel). Это рабочая схема для теста, для постоянного использования – фрустрирует
– **Бесплатные tier'ы облаков**: AWS / GCP / Azure дают $300 кредит на первые 90 дней. Хватает на gbrain VPS на квартал. Но настройка сложнее чем у Hetzner
– **Hetzner CPX22 (~6€/мес)** вместо CPX42: вдвое дешевле, 2 vCPU / 4 GB RAM. Хватает для gbrain если максимум 3 агента и низкая нагрузка

Если бюджет совсем впритык – начни с локального Docker + Cloudflare Tunnel, потом перебежишь на VPS.

## 6. Можно ли использовать другой LLM не Claude?

Технически модель – переменная, но весь стек заточен под Claude Code:

– Skills (этот репозиторий) пишутся в формате Claude Code SKILL.md – другие IDE / агентские системы их не понимают
– `dashi-plugin-claude-code` – plugin именно для Claude Code, не для OpenAI Assistant'ов
– `CLAUDE.md` – соглашение Anthropic для system prompt; в Cursor / Continue / Aider свои аналоги (`.cursorrules`, `CONVENTIONS.md`)

Если хочешь использовать GPT-4 / Gemini / локальную модель – придётся переписывать каждый компонент под нужный runtime. Это уже другой проект, не порт. Для начала остановись на Claude Code – он стабильнее всего держит длинные контексты и multi-agent workflow.

## 7. Безопасно ли self-host gbrain с моими данными?

Да – ты владелец сервера, и никакая третья сторона не видит содержимое:

– Postgres+pgvector живут на твоём VPS, никаких внешних API для хранения
– Embeddings считаются локально (FastEmbed multilingual) либо через OpenAI Embeddings API – выбираешь сам в конфиге
– MCP серверы доступны только через Bearer auth с твоим токеном; токены sha256-захешированы в БД
– Бэкапы (если настроишь) идут в твой собственный bucket / restic repo

Что важно сделать:

– Закрыть `ufw allow` только на нужные порты, остальное – DROP
– SSH только по ключу, password-auth disabled
– HTTPS через Let's Encrypt + Caddy
– Регулярные бэкапы (`pg_dump` + restic – см. документацию gbrain)

С этими настройками self-host gbrain безопаснее чем любой облачный SaaS для AI-памяти.

## 8. Что если я хочу 5 агентов вместо 3?

Просто повторяешь lesson для каждого нового агента. Лимиты:

– Anthropic Max выдерживает несколько параллельных Claude Code сессий (точные числа меняются – проверяй в текущих docs Anthropic)
– gbrain Postgres на CPX42 (4 GB RAM) спокойно держит 10+ агентов с активной memory writing
– Telegram – по одному боту на агента, лимита на боты у твоего аккаунта по сути нет

Что меняется при росте:

– Координация усложняется – нужен явный координатор-агент (роль), который раздаёт задачи
– Cost остаётся на уровне Max + VPS, агенты – это разные workspace'ы внутри одного Claude Code account
– Памяти gbrain потребуется больше – на 10 агентов рекомендую CPX52 (8 vCPU / 16 GB)

## 9. Где брать TOV (Tone of Voice) для marketer?

TOV – Tone of Voice, голос твоего бренда. Два пути:

– **Свой собственный**: описываешь в template marketer'а под `## TOV` секцией. Тон, лексика, что нельзя писать, примеры из твоих текущих постов. Это самый качественный путь – marketer пишет твоим голосом, а не «корпоративно-вежливым»
– **Сгенерировать sample**: попроси Claude написать 5 пробных постов в нужной нише, отредактируй вручную как захочется, скорми обратно в template как «вот эталоны». Marketer будет ориентироваться на них

Главное – TOV должен быть текстовым правилом, не картинкой. Чем подробнее – тем меньше правок в готовых постах потом.

## 10. Что если хочу англоязычный лесон?

На текущий момент уроки и шаблоны только на русском. Английская версия в планах, но точных сроков нет. Что можно сделать сейчас:

– Перевести шаблоны вручную (Claude сам прекрасно справится – дай ему файл и попроси перевод с сохранением структуры)
– Использовать английский внутри агентов: SOUL.md / `CLAUDE.md` сами по себе language-agnostic, в них пишешь на любом языке
– Skills (senior-brainstorm, telegram-bot-builder) уже работают с любыми языками – ты задаёшь на каком общаться

Если сделаешь свой английский перевод и хочешь поделиться – открой PR, добавим в репозиторий как `lessons/lesson-3-agents-with-gbrain/en/`.
