# Templates

Готовые шаблоны для запуска агентов на базе Claude Code. Каждый шаблон – рабочая заготовка `CLAUDE.md`, которую можно скопировать в workspace нового агента, подставить значения вместо `{{плейсхолдеров}}` и сразу получить осмысленное поведение.

## Что внутри

### `claude-md/`

Три шаблона `CLAUDE.md` под типовые роли в команде агентов:

- **coder.md** – кодер / архитектор / ревьюер. Для агента, который пишет код, проектирует системы, делает code review.
- **marketer.md** – маркетолог / контент-креатор. Для агента, который ведёт каналы, генерирует контент, занимается лидогенерацией и unit-экономикой.
- **inbox-monitoring.md** – Second Brain (по методу Karpathy) и мониторинг группового чата. Для агента, который автоматически сохраняет всё входящее, компилирует знания в wiki и отвечает в групповых чатах.

Подробное руководство – в [`claude-md/README.md`](claude-md/README.md). Там список всех плейсхолдеров, правила интеграции с gbrain (4 MCP сервера) и пошаговая инструкция.

### `3-claude-md-templates-v2.html`

Визуальный гид по тем же трём шаблонам в виде одной HTML-страницы. Открывается в браузере, показывает структуру каждого шаблона, объясняет каждую секцию и помогает выбрать подходящий вариант под свою задачу. Удобно показать команде или приклеить в свой Second Brain.

## Quick start

1. Выбери шаблон под роль агента (например, `claude-md/coder.md`).
2. Скопируй его в workspace будущего агента:

   ```bash
   cp templates/claude-md/coder.md ~/.claude-lab/my-agent/.claude/CLAUDE.md
   ```

3. Открой скопированный файл и замени все `{{плейсхолдеры}}` на свои значения (имя агента, обращение к владельцу, имена соседних агентов, host своего gbrain и т. д.). Полный список плейсхолдеров – в [`claude-md/README.md`](claude-md/README.md).
4. Перезапусти Claude Code в новом workspace – он подтянет `CLAUDE.md` и будет вести себя в соответствии с шаблоном.

## Что добавлять самому

Шаблоны намеренно компактные. Под свой проект тебе нужно дописать:

- `core/USER.md` – кто владелец, как с ним общаться, в каком часовом поясе он работает.
- `core/rules.md` – операционные правила и границы (что можно автономно, что только с разрешения).
- `core/warm/decisions.md` – журнал решений по проекту.
- `core/TONE_OF_VOICE.md` – для маркетинг-агента, источник правды по голосу бренда.
- `.mcp.json` – конфиг с подключением к твоему gbrain (4 MCP сервера: `gbrain-memory`, `gbrain-recall`, `gbrain-swarm`, `gbrain-tasks`).

Все шаблоны написаны на русском, потому что рассчитаны на русскоязычного владельца. Если общаешься с агентом на другом языке – поменяй формулировки в SOUL-блоке, остальное останется рабочим.
