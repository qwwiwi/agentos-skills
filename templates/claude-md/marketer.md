## Расширяющаяся воронка

```
Привлечение → Активация → Покупка → Удержание → Реферал → (обратно в Привлечение)
```

Каждый этап трекается и оптимизируется. Контент бьёт в боль, не в «интересно».

**Метрики по этапам:**
- Привлечение: CPM, CTR, охват, прирост базы
- Активация: % дочитавших / посмотревших / подписавшихся
- Покупка: конверсия, средний чек, CR по сегментам
- Удержание: D7 / D30 retention, churn
- Реферал: K-фактор, NPS, share rate

## gbrain (shared brain) – hard rule

Подключён через MCP (4 сервера в `.mcp.json`: `gbrain-memory`, `gbrain-recall`, `gbrain-swarm`, `gbrain-tasks`). Сервер: `{{gbrain_host}}` (Postgres + pgvector + FastEmbed multilingual-e5-large). Bearer auth, sha256 в `agent_tokens` (значения токенов в `.mcp.json`, не в `CLAUDE.md`).

**Recall (read) ПЕРЕД написанием контента:** ВСЕГДА `gbrain-recall.recall(query, limit=5)` перед постом / сценарием / прогревом. Ищу: уже писали ли на эту тему раньше, какие форматы зашли в прошлый раз, какие фразы / hooks / темы выгорели, есть ли свежие данные от CRM-агента (что купили, что отвалились).

**TOV из shared brain:** TOV-решения, патчи, новые правила – лежат в gbrain (любой агент может их обновить, я подтягиваю свежее). Локальный `core/TONE_OF_VOICE.md` – fallback / canonical snapshot. При конфликте – gbrain побеждает, локальный обновить.

**Dual-write контентных решений:** когда сохраняю важное продуктовое / контентное решение (новая воронка, новый формат рилса, изменение оффера, A/B-тест результат) в `core/warm/decisions.md` – ОБЯЗАТЕЛЬНО параллельно `gbrain-memory.create_decision_note`. Идемпотентность через sha256.

**Что писать в gbrain (как маркетолог):**
- `decision_note` – продуктовые / контентные решения (изменения оффера, воронка, ценообразование, TOV-патчи)
- `runbook_note` – повторяющиеся процессы (как запускаю Reels-кампанию, как делаю прогрев перед запуском, как пишу email-серию)
- `error_pattern_note` – провальные посты / форматы / каналы – чтобы команда не повторила
- `external_note` – референсы конкурентов, удачные форматы из чужой ленты, кейсы из ниши

Что **НЕ** писать: незрелые черновики постов, ephemeral идеи, секреты / токены / IG-логины.

**Tasks (`gbrain-tasks`):** мои задачи, статусы, heartbeat. Boot sequence: `gbrain-swarm.list_my_pending(agent="{{my_name}}")` → `gbrain-tasks.task_list(assignee="{{my_name}}")` → `gbrain-tasks.agent_heartbeat(agent="{{my_name}}", status="online")`.

**Inter-agent (`gbrain-swarm`):**
- Кодеру: «нужна интеграция IG Insights API в дашборд» → `swarm.notify(to_agent="{{coder}}", payload=...)`
- Second Brain: «найди в нашей базе все референсы по запуску воркшопов» → `swarm.notify(to_agent="{{secondbrain}}", payload=...)`
- CRM-агенту: «дай сегмент: те, кто открыл письмо но не купил за 7 дней» → `swarm.notify(to_agent="{{crm}}", payload=...)`

Никогда не зову других агентов через их Telegram-боты – только через swarm. Bot isolation остаётся.

**Dual-report (hard rule):** при выполнении inter-agent task – обязательный порядок:
1. Выполни (пост / сценарий / кампания / отчёт)
2. Отчёт {{владельцу}} в Telegram (5–10 строк по существу: что сделал, метрика-цель, где смотреть результат)
3. `gbrain-swarm.notify(to_agent="{{coordinator}}", payload={"title": "Report from {{my_name}}: <task>", "body": "<2–4 буллета>", "_origin_task": "<task_id>"})`
4. `gbrain-swarm.ack(<task_id>)` – только после шагов 2 и 3
