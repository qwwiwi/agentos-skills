# Архитектура AgentOS – как 5 публичных репо собираются вместе

## TL;DR

AgentOS – это не один монолитный репозиторий, а набор из 5 независимых публичных репо, каждый из которых решает одну задачу. Этот репо (`agentos-skills`) даёт вам скиллы, темплейты и уроки. Остальные 4 репо – фундамент: workspace для агента, общая память, мост в Telegram и гайд по CLAUDE.md. Любой из них можно использовать самостоятельно, но вместе они образуют полноценную систему: команда из нескольких AI-агентов, которая помнит вашу историю, общается между собой и доступна вам через Telegram.

## Карта инфраструктуры

Все компоненты разложены по слоям. Каждый слой – отдельный репо.

| Слой | Назначение | Репо | Когда нужен |
|---|---|---|---|
| **Layer 0: Знания** | Скиллы, темплейты, уроки | `agentos-skills` (этот репо) | Всегда |
| **Layer 1: Workspace** | Память агента, hooks, semantic search | `public-architecture-claude-code` | На каждого агента |
| **Layer 2: Shared memory** | Общая память команды, MCP, inbox | `public-gbrain-agentos` | Когда 2+ агентов |
| **Layer 3: Telegram bridge** | Доступ к агенту через Telegram | `dashi-plugin-claude-code` | Когда нужен мобильный доступ |
| **Layer 4: CLAUDE.md гайд** | Как писать identity для агента | `edgelab-claude-md` | На этапе настройки |

## Зачем 5 репо вместо одного

Разделение даёт три практических преимущества:

1. **Можно собрать только то, что нужно.** Если вам достаточно одного локального агента без Telegram и без общей памяти – ставите только Layer 1. Один VPS на 2 GB RAM и хватит.
2. **Каждый слой обновляется отдельно.** Обновление workspace не ломает Telegram bridge. Обновление gbrain не требует переустановки агентов.
3. **Каждый слой имеет понятную ответственность.** Когда что-то ломается, понятно, в каком репо смотреть issue.

## Какой репо нужен в каком сценарии

| Сценарий | Layer 1 | Layer 2 | Layer 3 | Layer 4 |
|---|---|---|---|---|
| Один локальный coding-агент на ноутбуке | да | нет | нет | да |
| Один агент + Telegram доступ | да | нет | да | да |
| Команда из 3 агентов (coder + marketer + sales) на одном хосте | да (×3) | да | да (×3) | да |
| Только эксперимент со скиллами без агентов | нет | нет | нет | нет |
| Микро-сервер без shared memory, но с Telegram | да | нет | да | да |

## Порядок установки

Рекомендованный путь для урока «3 агента + общая память»:

1. Клонируете `agentos-skills` (этот репо) – получаете templates и lessons.
2. Устанавливаете `public-architecture-claude-code` install.sh – один раз на каждого агента (coder, marketer, sales).
3. Устанавливаете `public-gbrain-agentos` – один раз на отдельный VPS, обслуживает всех агентов.
4. Устанавливаете `dashi-plugin-claude-code` – один раз на каждого агента, привязывает к Telegram-боту.
5. В каждый workspace копируете соответствующий CLAUDE.md template из `agentos-skills/templates/claude-md/`.
6. Делаете smoke-test: один агент пишет другому через swarm.notify, второй получает сообщение в свой inbox.
